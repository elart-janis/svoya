# План разработки игры «Своя игра»

## 1. Цель

- Веб-игра для Zoom на HTML + Bootstrap 5.
- Поле: визуальная таблица `5x5`:
  - 5 строк категорий (в первом столбце).
  - 4 столбца стоимостей: `200`, `400`, `600`, `800`.
- Игроки: `1-15`.
- Счет игроков отображается в dropdown-блоке.

## 2. Текущий контракт данных

- `data/game.json`:
  - `values = [200, 400, 600, 800]`
  - `categories.length = 5`
  - в каждой категории 4 вопроса (по одному на каждую стоимость).
- `localStorage`:
  - ключ `svoyaGame.state`
  - `players`, `used`, `version`.

## 3. Этапы реализации

### Этап 1. Компактный верхний блок

Задачи:
- Минимизировать высоту верхней панели.
- Добавить кнопки `+` / `-` игроков.
- Показать текущее количество игроков (`x/15`).
- Вынести список игроков и текущий счет в dropdown.

Готово, если:
- Верхняя панель компактная и все элементы управления доступны.

### Этап 2. Поле 5x5 с новой ориентацией

Задачи:
- Перестроить рендер таблицы:
  - первый столбец категории;
  - горизонтально стоимости `200/400/600/800`.
- Обновить валидацию JSON под 5 категорий и 4 стоимости.

Готово, если:
- Поле визуально соответствует формату `5x5`.

### Этап 3. Игроки 1-15 и сохранение

Задачи:
- Реализовать добавление/удаление игроков в диапазоне `1-15`.
- Разрешить редактирование имен игроков в dropdown.
- Сохранять изменения в `localStorage`.

Готово, если:
- После перезагрузки состав игроков, имена и счет восстанавливаются.

Статус: `выполнено`.

### Этап 4. Игровой цикл вопроса

Задачи:
- Открытие вопроса из ячейки поля.
- Переключение `Вопрос` -> `Ответ`.
- Фиксация `Верно` / `Неверно` с изменением счета.
- Блокировка сыгранных ячеек.

Готово, если:
- Полный цикл вопроса работает без ручного редактирования счета.

Статус: `выполнено`.

### Этап 5. Сброс и стабилизация

Задачи:
- `Сбросить игру` с подтверждением.
- Очистка `localStorage` и возврат стартового состояния.
- Финальная проверка интерфейса под Zoom.

Готово, если:
- Сброс предсказуем, интерфейс читаем на демонстрации.

Статус: `выполнено`.

## 4. Definition of Done

- [x] Поле отображается как `5x5` (категория + 4 стоимости).
- [x] Игроки управляются в диапазоне `1-15`.
- [x] Счет виден в dropdown-блоке.
- [x] Данные валидируются и загружаются через `fetch()`.
- [x] Состояние сохраняется и восстанавливается.
- [x] `Сбросить игру` работает с подтверждением.
