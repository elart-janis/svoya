# Инструкция для ИИ по генерации `game.json`

## Цель

Сгенерировать новый файл `data/game.json` для игры «Своя игра» в текущем формате проекта.

## Жесткий контракт JSON

ИИ должен вернуть **только валидный JSON-объект** без пояснений, без markdown, без комментариев.

Структура:

- Корень:
  - `title`: строка
  - `values`: массив строго `[200, 400, 600, 800]`
  - `categories`: массив из **ровно 5** категорий
- Категория:
  - `id`: уникальный slug в lower_snake_case или kebab-case на латинице (например `space`, `world-history`)
  - `name`: название категории (строка, обычно на русском)
  - `questions`: массив из **ровно 4** вопросов
- Вопрос:
  - `id`: уникальный идентификатор, рекомендуется шаблон `${categoryId}_${value}`
  - `value`: одно из значений `200 | 400 | 600 | 800`
  - `question`: строка с текстом вопроса
  - `answer`: строка с текстом ответа

## Ограничения контента

- Только текст (без HTML, markdown, ссылок, эмодзи).
- Язык: русский.
- В каждой категории должны быть вопросы для всех 4 стоимостей: `200`, `400`, `600`, `800` (без пропусков и дублей).
- Все `id` категорий и вопросов уникальны.
- Вопросы и ответы должны быть корректными и понятными детям.
- Избегать двусмысленных, спорных и устаревших формулировок.

## Требования к качеству

- 5 категорий должны быть тематически разными.
- Сложность вопросов должна возрастать от `200` к `800`.
- Ответы короткие и однозначные (обычно 1 фраза).
- Не повторять один и тот же факт в разных категориях.

## Формат ответа от ИИ

Обязательно:

1. Верни только JSON.
2. Не добавляй текст до/после JSON.
3. Не оборачивай JSON в тройные кавычки.

## Готовый промпт для генерации

Скопируй и отправь ИИ:

```text
Сгенерируй новый data/game.json для детской викторины «Своя игра».
Верни только валидный JSON без пояснений и без markdown.

Строгий формат:
- title: строка
- values: строго [200, 400, 600, 800]
- categories: ровно 5 категорий
- каждая категория: id (уникальный slug на латинице), name (строка), questions (ровно 4)
- каждый вопрос: id (уникальный), value (200|400|600|800), question (строка), answer (строка)
- в каждой категории должны присутствовать все стоимости: 200, 400, 600, 800

Контент:
- только текст, язык русский
- вопросы понятные детям
- сложность растет от 200 к 800
- ответы короткие и однозначные
- без повторов фактов между категориями
```

## Проверка после генерации (локально)

После вставки JSON в `data/game.json` можно проверить контракт:

```bash
node --input-type=module -e "import { validateGameData } from './assets/js/data.js'; import fs from 'node:fs'; const data = JSON.parse(fs.readFileSync('./data/game.json','utf8')); validateGameData(data); console.log('validation ok');"
```
